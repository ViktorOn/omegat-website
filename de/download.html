Die "Standard"-Version von OmegaT beinhaltet ein vollständiges Benutzerhandbuch und wird für gewöhnlich einige Zeit getestet worden sein, um Fehler zu beseitigen. Es ist die empfohlene Version für Erstbenutzer.

Wie der Name schon sagt, ist die "neueste" Version von Omegat aktueller. Wie auch immer, das Benutzerhandbuch könnte nicht mehr auf dem aktuellen Stand (und deshalb möglicherweise verwirrend) sein und einige kleinere Fehler sind möglich. Nichtsdestotrotz, die "Neueste"-Versionen von OmegaT sind gewöhnlich sehr stabil und werden für Benutzer empfohlen, die schon einige Erfahrung mit OmegaT haben.

Voraussetzungen

Systemvoraussetzungen

OmegaT wird auf jedem System laufen, auf dem eine passende JRE (Java Runtime Environment) installiert worden ist oder installiert werden kann. Die JRE wird jetzt mit OmegaT geliefert und muss nicht separat erworben werden. OmegaT wurde erfolgreich unter Windows (alle Versionen ab 98), Macintosh OS X und Linux installiert.

Dateien im Microsoft Office XML-Format können nun direkt in OmegaT übersetzt werden. Dateien in älteren MS Word-, Excel- und Powerpoint-Formaten können nach der Konvertierung in die entsprechenden aktuellen Microsoft Office-Formate oder alternativ zum Office Open XML-Format, das in OpenOffice.org oder LibreOffice (ebenfalls kostenlos) durchgeführt werden kann, übersetzt werden. Um OpenOffice.org oder LibreOffice zu erhalten, folgen Sie dem Link "Software von Drittanbietern" auf der linken Seite.

Benutzeranforderungen

OmegaT ist nicht besonders schwierig zu installieren oder zu verwenden, aber es erfordert die Bereitschaft, die Anweisungen zu lesen und zu befolgen. Wenn Sie eine natürliche Abneigung gegen das Lesen von Anweisungen haben und Ihr Ansatz zu neuer Software darin besteht, auf jeden Knopf zu klicken, den Sie sehen, bis etwas dem gewünschten Effekt ähnelt, dann ist OmegaT wahrscheinlich nicht für Sie geeignet.

Legacy-Versionen

Sie können immer noch ältere, abgelöste Versionen von OmegaT finden.

Legacy-Versionen sind ältere Versionen, die abgelöst worden sind. Für die meisten Benutzer sind sie uninteressant. Einige Benutzer können jedoch Einschränkungen (durch ihre Betriebssysteme oder ihre Java-Version) ausgesetzt sein, die zu einer Bevorzugung einer älteren Version führen können.

Mit / ohne JRE

Damit OmegaT ausgeführt werden kann, muss Java Runtime Environment (JRE) auf Ihrem Computer installiert sein.

Wenn Sie sicher sind, dass eine geeignete Version der JRE bereits auf Ihrem Computer installiert ist, können Sie eine Version von OmegaT ohne die JRE herunterladen. Der Download wird schneller sein und die Installation benötigt weniger Speicherplatz auf Ihrer Festplatte. Entscheiden Sie sich in diesem Fall für die Standardversion oder neueste Version und wählen Sie dann eine Version mit der Kennzeichnung "ohne Java" aus.

Wenn Sie die OmegaT-Version mit JRE installieren, obwohl die JRE bereits auf Ihrem Computer installiert ist, wird die JRE von OmegaT separat installiert und steht nicht in Konflikt mit ihr. Es ist daher immer sicher, die Version von OmegaT mit JRE zu installieren. Wenn Sie sich unsicher sind, ob Sie eine geeignete Version der JRE haben oder nicht, wählen Sie eine Version mit JRE (oder die Standardversion).

Mac OS X-Benutzer: Die JRE wird standardmäßig auf Mac OS X installiert. Es gibt daher keine Version für Mac OS X mit der JRE.

Linux-Benutzer: Die meisten Linux-Distributionen werden mit einer Java-Laufzeitumgebung ausgeliefert, einige Open-Source-Implementierungen funktionieren jedoch nicht ordnungsgemäß mit OmegaT (obwohl dies möglicherweise nicht sofort offensichtlich ist). Wenn Sie bei Linux relativ neu sind, empfehlen wir Ihnen, eine OmegaT-Version mit der JRE zu wählen. Erfahrenere Linux-Benutzer mögen es vorziehen, Oracle (früher Sun) JRE auf Systemebene zu installieren.

Signiert / Unsigniert

Mac OS X verfügt über eine Funktion namens Gatekeeper, die die Sicherheit erhöht, indem es das Ausführen nicht signierter Programme erschwert. OmegaT für Mac OS X ist sowohl in signierter als auch in unsignierter Version verfügbar. Sie sind funktional identisch, aber die signierte Version wird bevorzugt, da sie vom System als von einem bekannten Entwickler veröffentlicht erkannt wird.

Es kann zu Verzögerungen bei der Veröffentlichung signierter Versionen kommen. Eilige Benutzer können stattdessen die unsignierte Version verwenden; Wenn Sie Probleme beim Starten der App haben, lesen Sie, wie Sie eine App von einem unbekannten Entwickler öffnen.

Die Webstart-Version bietet eine alternative Möglichkeit, OmegaT auf Ihrem Computer zu installieren.

Mit dieser Version werden zukünftige Updates automatisch installiert.

Je nach Betriebssystem erhalten Sie während der Installation möglicherweise mehrere Sicherheitswarnungen.
Das Zertifikat ist von PnS Concept Sarl signiert, einer in Frankreich registrierten Firma.

Die Berechtigungen, die Sie dieser Version erteilen (die als "uneingeschränkter Zugriff auf den Computer" bezeichnet werden können) unterscheiden sich nicht von den Berechtigungen, die Sie der lokalen Version zuweisen, zum Beispiel einem Zugriff auf die Festplatte Ihres Computers.

Um die Webstart-Version zu installieren, müssen Sie bereits Java Version 1.3 oder höher auf Ihrem Computer installiert haben.

Informationen zum Datenschutz: OmegaT Java Webstart speichert keine Ihrer Daten auf den Webservern von OmegaT. Die Anwendung läuft auf Ihrem Computer. Ihre Dokumente und Translation Memory verbleiben auf Ihrem Computer und das <a2>OmegaT</a2>-Projekt wird keinen Zugriff auf Ihre Arbeit oder Information haben.

Die /trunk (Alpha)-Version spiegelt den neuesten Stand der Entwicklung wider. Sie wird nur ernsthaften Entwicklern und Testern empfohlen. Sie kann auch mit diesem Subversion-Befehl erhalten werden:

svn checkout svn://svn.code.sf.net/p/omegat/svn/trunk omegat-svn

Periodische Builds sind bei Sourceforge verfügbar.

Quellcode-Pakete werden von Benutzern nicht benötigt und sind nur für Entwickler gedacht. Sie finden sie in den Ordnern der entsprechenden kompilierten Versionen.
